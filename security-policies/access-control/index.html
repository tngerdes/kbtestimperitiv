    Security Policy #6: Access Control

    Purpose of Policy

    The purpose of this policy is to assure that systems containing PII and/or sensitive company data are accessed only by those persons or software programs that have been granted appropriate access rights.

    Unique User Identification

    • Employees will be assigned a unique user identification (user ID) to access any system or application that transmits, receives, or stores PII and/or sensitive company data.
    • Each employee must ensure their assigned user ID is appropriately protected and only used for legitimate access.
    • If an employee believes their user ID has been compromised, they must report the security incident immediately.

    Password Requirements

    Employees should be aware of the following procedures to create strong passwords:

    • Minimum of eight (8) characters in length.
    • Must incorporate both upper and lower case letters (a–z and A–Z).
    • Must incorporate digits and punctuation characters (0–9, `! @ # $ % ^ & * ( ) _ - + = { } [ ] : ; " ' | \ / ? < > , . ~ ``).
    • Must not be words found in a dictionary.
    • Must not include easily guessed personal information (names, pets, birth dates, etc.).

    Password Protection Procedures

    • Passwords must not be written down.
    • Passwords must not be shared with other employees.
    • If an employee suspects their password has been compromised, they must report the incident immediately.
    • Passwords must be changed at least every 90 days, unless Multi-Factor Authentication (MFA) is in use and enforced.
    • After a number of failed password attempts, the employee's account will be disabled.

    Automatic Logoff

    • Systems accessing or storing PII and/or sensitive company data should implement automatic logoff after a determined period of inactivity (e.g., 10 minutes).
    • When leaving a workstation, server, or other computer system unattended, employees must lock the system (e.g., CTRL + ALT + DELETE > Lock) or log out of all applications and systems containing PII.

    Encryption and Decryption

    • To the extent technically feasible, all portable devices containing PII and/or sensitive company data should be encrypted.
    • Encryption should be used when sending any PII across public networks and wireless networks, including email and the Internet.
    • Employees should be trained on the use of encryption to protect PII and sensitive company data.
    • All backup tapes and media containing PII and/or sensitive company data should utilize encryption.
    • Secure encrypted remote access procedures should be implemented for systems accessing or storing PII. Examples include VPN clients, authenticated SSL web sessions, and encrypted Citrix/RDP client access.
    • Two-factor authentication (MFA) (e.g., SMS PIN notification) should be implemented where technically feasible.
    • All wireless access to networks should utilize encryption mechanisms.
    • Employees should not utilize open public Wi-Fi networks.
