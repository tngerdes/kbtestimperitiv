# Security Policy #8: Equipment Disposal

## Purpose of Policy

All media containing PII and sensitive company data will be disposed of in a manner that **destroys the data** and does not allow unauthorized access.

---

## Procedures for Computer/Hardware Disposal

1. The Security Officer or delegate will **notify IT** of equipment that needs to be disposed of.
2. The Security Officer or delegate will **determine the data sensitivity** of the data to be disposed of (see Data Classification Table below).
3. IT will assess the condition of the equipment and:
   - **Track the disposal** of the device (type of hardware, serial number, etc.) — See *Appendix A: Media Disposal Log*.
   - Run **approved wiping software** on all devices to ensure all PII and sensitive company data is removed.
   - This may include **physical destruction** (see Methods of Destruction below).
   - **Verify** that the hardware's data has been removed.
   - **Dispose of the hardware**.
4. The Security Officer or delegate/IT will **document the destruction** and retain the record — See *Appendix A: Media Disposal Log*.
5. If taken to an outside facility — The media shall be taken to an **approved, certified facility** for erasure or destruction. A **letter of certification** regarding the date and time of erasure/destruction shall be obtained.

---

## Data Classification Table

| Classification | Description | Disposal Requirement |
|---|---|---|
| **Low (Unclassified)** | Basic operating system, personal files, etc. | No requirement to erase, but normally erase using any means such as reformatting or degaussing. |
| **Medium (Sensitive but not Confidential)** | Business-related information not considered sensitive company data. | Erase using any means such as reformatting or degaussing. |
| **High (Confidential)** | PII and sensitive company data. | Data must be erased using an approved technology to ensure it cannot be read using special forensic techniques. |

---

## Examples of Hardware Devices Covered

- Workstations and laptops
- Tablets (iPad/Android)
- Smartphones
- Server hard drives
- Memory sticks (USB drives)
- CD-ROM/DVD-ROM discs
- Storage/Backup tapes
- Hard drives
- Copiers/Scanners/Fax machines
- Any equipment that contains PII or sensitive company data

---

## Methods of Destruction

| Method | Description |
|---|---|
| **Clear** | Use software or hardware products to overwrite storage space with non-sensitive data. This process replaces written data with random data. Cannot be used for media that is damaged or not rewriteable. |
| **Purge** | Degaussing or executing the firmware Secure Erase command (for ATA drives only). Degaussing exposes magnetic media to a strong magnetic field to disrupt recorded magnetic domains. Effective for damaged, inoperative, or large-capacity media. |
| **Destroy** | Physical destruction of media to render it completely unreadable and unrecoverable. |

---

## Appendix A — Media Disposal Log

All disposed/destroyed media must be logged. The record should include:

- **Date of Destruction**
- **Authorized By**
- **Description of Information Disposed of or Destroyed** (Manufacturer/Model/Serial Number/etc.)
- **Backup of PII or sensitive company data?** (Required if this is the only copy) — Yes / No / If Yes, list backup location
- **Method of Destruction:** Clear / Purge / Destroy
- **Destruction Method Used**
- **Final Disposition of Media:** Disposed / Reused Internally / Reused Externally (sold/donated/etc.) / Returned to Manufacturer/Leasing Company/Vendor / Other

> **Save this log and retain indefinitely. Upload to the Security Portal.**
